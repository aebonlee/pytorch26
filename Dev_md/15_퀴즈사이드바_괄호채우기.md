# 15. 퀴즈 사이드바 재디자인 · 프로젝트 괄호채우기 (2026-07-17)

## 1) 복습퀴즈 사이드바 (대표 지정 디자인)

기존 "문항 미리보기 목록" → 아래 3단 구성으로 교체:

1. **1~15 번호 버튼 그리드** (5×3, 정사각) — 풀이 상태를 색으로:
   - 미풀이=회색 테두리 / 풀이함=파랑 / 채점 후 정답=초록·오답=빨강
   - 클릭 시 해당 문항으로 스크롤, 아래 색 범례 표시
2. **점수 박스** — 진행 중엔 "응답 n/15", 채점 후엔 "이번 점수 n/15" 크게
3. **응시 기록 박스** — 1~3회차 점수(미응시는 —) + 남은 기회 / 응시 완료

CSS: .quiz-nums(그리드), .qnum-btn(+answered/correct/wrong), .quiz-side-box, .quiz-legend

## 2) 프로젝트 코드 버튼 2종 (괄호채우기 + 완성형)

- 8개 프로젝트 전부에 `skeleton`(fill01~08) 추가 — 완성형과 같은 구조에서
  **핵심 수식·정의만 ________ 빈칸** 처리 (각 빈칸에 번호·힌트 주석)
  - fill01 Q-Learning: greedy 행동·TD 업데이트·ε 감쇠·평가 행동 (4칸)
  - fill02 Taxi: SARSA/QL 목표 차이·업데이트 (2칸)
  - fill03 DDQN: 선택/평가 분리·TD 목표·타깃 동기화 (4칸)
  - fill04 A2C: n-step 리턴·어드밴티지·손실 구성 (3칸)
  - fill05 DDPG: TD 목표·Actor 손실·소프트 업데이트 (3칸)
  - fill06 TAC: q-log 식·수정 지점 2곳 (3칸)
  - fill07 MountainCar: shaping 보상·curiosity 보상·TD 목표 (3칸)
  - fill08 커스텀 환경: 공간 정의 2·보상·종료 조건·Q 업데이트 (5칸)
- 카드 우하단 버튼 2개: "🧩 괄호채우기 보기"(회색) / "📄 완성형 코드 보기"(오렌지 톤, .sol-full)
- 각각 독립 토글 — 빈칸을 풀다가 완성형과 비교 가능. 기본은 둘 다 닫힘
