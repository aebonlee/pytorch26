# 2026-07-20 · 모바일 헤더 메뉴 최적화 (2줄 줄바꿈 + 햄버거)

## 문제

모바일에서 상단 메뉴가 깨져 보임.

원인은 단순하다. 헤더 메뉴가 **12개**(About·선수학습·환경설정·1~3일차·요약정리·실습소스·프로젝트·복습퀴즈·심화학습·VOD)인데
`.nav`가 `display:flex`에 `flex-wrap` 없이 한 줄로만 놓여 있었고,
`.site-header .inner`는 `height: 60px` **고정**이었다.
좁은 폭에서 메뉴가 로고를 밀어내거나 헤더 밖으로 삐져나온다.
기존 대응은 `@media (max-width:640px)`에서 글자·패딩을 줄이는 것뿐이라 근본 해결이 아니었다.

## 조치 — 폭에 따라 2단계로 접는다

| 폭 | 동작 |
|---|---|
| 1101px~ | 기존 그대로 한 줄 |
| ~1100px | 메뉴가 **2줄로 줄바꿈**, 헤더 높이 가변(`height:auto; min-height:60px`) |
| ~760px | 메뉴 숨김 + **햄버거 버튼**, 열면 **2열 그리드 드롭다운** |

### src/components/Layout.jsx

- 메뉴 12개를 `MENU` 배열로 뽑고 `map`으로 렌더 (JSX 12줄 나열 → 배열 1곳 관리)
- `open` 상태 추가. 햄버거 버튼은 `.nav-toggle`, 막대 3개는 `<i>` 3개로 그리고 열리면 X자로 회전
- **테마 토글은 드로어 안에 넣지 않고** `.header-actions`로 빼서 헤더 바에 계속 노출.
  이유: 모바일에서 테마 전환하려고 메뉴를 여는 건 불필요한 한 단계다
- 닫히는 경로 4가지: 메뉴 클릭(`<nav>` onClick) / 경로 변화(`useLocation`) / ESC / 스크림 탭
  - 경로 변화만으로는 **같은 메뉴 재클릭** 때 안 닫힌다 → onClick을 함께 둠
  - 반대로 onClick만으로는 로고 클릭 이동이 안 잡힌다 → 둘 다 필요
- `aria-expanded` / `aria-controls="site-nav"` / `aria-label` 부여

### src/styles/global.css

- `.header-actions`, `.nav-toggle`, `.bars` 신규
- `@media (max-width:1100px)` — `.nav { flex-wrap: wrap }` + inner `height:auto`
- `@media (max-width:760px)` — `.nav`를 `position:absolute; top:100%`로 헤더 아래 드롭다운,
  `grid-template-columns: 1fr 1fr`
- 기존 640 블록에 있던 `.nav a` 축소 규칙은 1100 블록으로 흡수(중복 제거), `.logo .sub` 숨김은 유지

## 밟은 함정 (같은 실수 반복 방지)

- **`height:60px` 고정을 안 풀면 줄바꿈해도 2번째 줄이 헤더 밖으로 나간다.** `flex-wrap`만 넣고 끝내면 안 된다.
- **드롭다운 배경에 `--header-bg`를 쓰면 안 된다.** 이 값은 반투명(0.92)이라 뒤 본문 글자가 비쳐 읽히지 않는다. 불투명 `--bg` 사용.
- 드롭다운을 `position:absolute`로 헤더에 붙일 수 있는 건 `.site-header`가 `position:sticky`(=위치 지정 요소)이기 때문. 이 sticky를 풀면 드롭다운이 페이지 최상단으로 날아간다.
- 스크림 `z-index:40` — 헤더(50)보다 **낮아야** 한다. 높으면 메뉴 자체가 스크림에 덮여 클릭이 안 먹는다.
- 화면 폭 판정은 **CSS 미디어쿼리에만** 맡겼다. JS로 `window.innerWidth`를 재면 리사이즈·화면 회전 때 상태가 어긋난다. `open`은 "열려는 의도"만 뜻하고 데스크톱에선 CSS가 `.nav`를 항상 보이게 해서 무해하다.
- `.bars i`의 transition을 `prefers-reduced-motion` 블록에 추가.
  이유: 이 사이트는 2026-07-17에 Windows 애니메이션 끄기 설정에서 화면이 멎은 이력이 있다(`/check.html` 진단 페이지 참고). 새 애니메이션을 넣을 때마다 이 블록도 같이 갱신한다.
