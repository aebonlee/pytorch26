# 16. 괄호채우기 난이도 1·2·3 선택형 (2026-07-17)

## 대표 지시

괄호채우기 개수를 난이도별 1, 2, 3으로 구분해 선택할 수 있게.

## 구현 — 정적 소스 24벌 대신 런타임 생성

- projects.js의 skeleton 소스 8벌 삭제 → `blanks: [{ level, find, hint, nth? }]` 스펙으로 교체
  - find = solution.source 안의 정답 코드 조각(정확 일치), 난이도 L 선택 시 level<=L **누적** 적용
  - nth = 같은 조각이 여러 번일 때 몇 번째를 비울지 (예: #3 타깃넷 동기화는 nth:2 — 1번째는 초기화 코드라서)
- `utils/skeleton.js`: makeSkeleton(솔루션, blanks, 난이도) → 등장 순서대로 `____①____` 치환 + 번호 힌트 목록 반환
- UI: 괄호채우기 패널 상단에 난이도 탭(1/2/3, 빈칸 개수 표시) + "빈칸 힌트" 박스 + 코드
- 파일명은 fill0N_lv{난이도}.py로 자동 생성

## 유지보수 주의

**solution 코드를 수정하면 blanks의 find도 함께 맞출 것** — find가 어긋나면 해당 빈칸만 조용히 사라진다 (에러 없음). 확인 명령:
```bash
node --input-type=module -e "import p from './src/data/projects.js'; import {makeSkeleton,blankCount} from './src/utils/skeleton.js'; p.forEach(x=>console.log(x.id, [1,2,3].map(l=>makeSkeleton(x.solution.source,x.blanks,l).hints.length+'/'+blankCount(x.blanks,l)).join(' ')))"
```
