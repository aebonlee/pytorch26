# 01. 초기 구축 (2026-07-17)

## 개요

멀티캠퍼스 공개과정 "PyTorch로 배우는 강화학습" 3일(21시간) 학습사이트를 빈 리포에서 신규 구축.

## 설계 결정

- **인증·Supabase 없음**: 7명 소규모 공개과정이라 로그인 불필요. 진도체크는 localStorage(`pytorch26_progress`)로 처리 → Supabase key fallback 이슈 원천 차단.
- **HashRouter 사용**: GitHub Pages 프로젝트 페이지(aebonlee.github.io/pytorch26/)에서 새로고침 404를 피하기 위함. 404.html 트릭 불필요.
- **콘텐츠 = 데이터 분리**: 21교시 전체(이론 요약 + 실습 코드)를 `src/data/day1~3.js`에 데이터로 분리. 페이지 컴포넌트는 렌더링만 담당.
- **마지막날 퀴즈+심화 패턴 적용**: 복습퀴즈 15문항(`quizzes.js`) + 도전과제 4단계·추천자료 12종(`extrastudy.js`). 3일차 7교시 페이지 하단에서 퀴즈/심화로 연결.

## 콘텐츠 구성 (멀티캠퍼스 공식 목차 그대로 21교시)

- Day 1 (Tabular): RL 소개(밴딧 ε-greedy) → MDP(GridWorld 정의) → DP → PI/VI 구현 → MC/TD → SARSA/Q-Learning 소개 → CliffWalking 구현
- Day 2 (Value/Policy-based): DQN(ReplayBuffer) → Double DQN → PyTorch 기초(학습루프 5단계) → DQN CartPole 구현(DOUBLE 스위치) → Policy Gradient(REINFORCE) → Actor-Critic → A2C 구현
- Day 3 (Advanced AC): DDPG 소개(Actor/Critic/soft_update) → DDPG Pendulum 구현 → Max Entropy RL → SAC(GaussianActor·rsample) → TAC(Tsallis q-log) → SAC 구현 → TAC 구현

코드는 세션 간 재사용 흐름(ReplayBuffer→DQN→DDPG→SAC→TAC)으로 설계 — 앞 교시에서 만든 클래스를 뒤 교시가 이어 씀.

## 기술 스택

React 19 + Vite 8 (JSX, TS 아님), react-router-dom 7 (HashRouter), 의존성 최소화(하이라이터·Tailwind 없음, CSS 커스텀 프로퍼티 다크 테마, PyTorch 오렌지 #EE4C2C 시그니처).

## 배포

- GitHub Actions (`deploy.yml`): main 푸시 → build → actions/deploy-pages. Pages build_type=workflow로 전환함 (legacy 아님 — 되돌리면 빈 화면).
- vite base `/pytorch26/`. 커스텀 도메인 전환 시 base를 `/`로 바꿔야 함 (rest02/03/04에서 반복된 자산 404 버그).
