# 2026-07-19 구형 데스크톱 크롬 빈 화면 — 빌드 타깃 하향

## 증상
- 대표 맥북에서는 정상, 다른 데스크톱(크롬)에서는 메인페이지가 멈춘 듯 JS가 동작하지 않음.
- 해당 데스크톱 콘솔에 에러 없음. 같은 데스크톱에서 dsu(동신대) 사이트는 정상 동작.

## 원인 진단
- 배포 자체는 정상 확인: 라이브 HTML 정상, JS 번들 200 응답(498KB), localStorage 접근은 try/catch 방어 완료.
- dsu는 Vite 7(타깃 baseline: Chrome 107+), pytorch26은 Vite 8(타깃 baseline이 더 최신).
  → 두 타깃 사이 버전의 구형 크롬(예: Windows 7/8에서 멈춘 Chrome 109)에서 dsu는 되고 pytorch26만 실행 불가한 증상과 일치.
- CSS 신문법(중첩)은 소스에 없음 → JS 문법 차이로 판단.

## 조치
- `vite.config.js`에 build.target / cssTarget을 `['chrome87','edge88','firefox78','safari14']`로 명시(Vite 구버전 기본값 수준).
  이유: 강의장(7/27~29 역삼 멀티캠퍼스) PC 사양을 통제할 수 없으므로, dsu가 도는 브라우저면 반드시 pytorch26도 돌게 보장.
- `npm run build` 검증 통과(498.93 kB, 269ms) 후 커밋·푸시 → GitHub Actions 자동 배포.

## 남은 일
- 문제 데스크톱에서 강력 새로고침(Ctrl+Shift+R) 후 정상 동작 확인 — 대표 직접 확인.
- 그래도 재현되면 해당 크롬 버전(chrome://version)과 Network 탭의 index-*.js 상태 코드 확보 후 재진단.
