# 02. 커스텀 도메인 전환 (2026-07-17)

## 결과

- 공개 URL: **https://pytorch26.dreamitbiz.com/** (정상 서빙 확인 — 페이지 200, JS 자산 200)
- aebonlee.github.io/pytorch26 은 커스텀 도메인으로 리다이렉트됨

## 빈 화면의 원인 2가지 (재발 방지용 기록)

1. **legacy Jekyll 빌드가 소스를 서빙**: 초기 배포 때 workflow 배포 성공 직후 legacy `pages build and deployment`가 뒤늦게 완료되며 리포 소스(index.html의 `/src/main.jsx`)를 그대로 덮어씀 → 브라우저가 JSX를 실행 못 해 빈 화면.
2. **커스텀 도메인 등록 시 build_type이 legacy로 리셋**: `gh api ... -f cname=...` 등록 후 GitHub이 "Create CNAME" 커밋을 만들면서 build_type이 workflow → legacy로 되돌아감 (농협사료 때와 동일 증상). workflow로 재전환 + `gh workflow run deploy.yml` 재트리거로 해결.

## 변경 사항

- vite base: `/pytorch26/` → `/` (커스텀 도메인에서 base가 리포명이면 자산 404)
- package.json homepage, index.html og:url, README 배포 URL → pytorch26.dreamitbiz.com
- DNS는 Cloudflare 와일드카드로 이미 해석되고 있어 별도 DNS 작업 불필요했음

## 점검 명령

```bash
gh api repos/aebonlee/pytorch26/pages --jq '{cname, build_type, status}'
# build_type이 "workflow"가 아니면 빈 화면 → PUT으로 재전환 후 deploy.yml 재실행
```
